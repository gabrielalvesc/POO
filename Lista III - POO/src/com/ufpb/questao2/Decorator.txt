Ele é definido como aberto-fechado, sendo aberto para expansão e fechado para alteração.  É utilizado para adicionar, em tempo de execução, novas "funções". O decorator é um padrão do tipo estrutural e funciona como um wrapper para classes existentes. Este padrão cria uma classe decoradora que empacota a classe original para prover novas funcionalidades. Tudo isto é feito sem que haja alteração na estrutura da classe. No decorator temos, uma interface do componete que será "decorado", o componente a ser "decorado" que implementa a interface, uma classe abstrata "decorator" que também implementa a interface, e as classes "decoradoras" que extendem a clase "decorator".
